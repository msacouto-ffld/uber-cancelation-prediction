## Problem Statement

Ride-hailing platforms face significant operational inefficiencies when bookings are cancelled. Cancellations create customer dissatisfaction, driver idle time, and reduced platform efficiency. This project builds a machine learning model that **predicts ride cancellations at booking time**, using only features available before the ride begins, so the platform can intervene proactively.

**Target variable:** `Cancelled = 1` if Booking Status ≠ Completed, else 0  
**Primary metrics:** ROC-AUC, Recall (Cancelled class)

---

## Results

| Model | ROC-AUC | Recall (Cancelled) | F1 (Cancelled) |
|---|---|---|---|
| Logistic Regression | 0.609 | 0.183 | 0.310 |
| Random Forest | 0.687 | 0.448 | 0.504 |
| **Gradient Boosting** | **0.725** | **0.666** | **0.581** |

Best model: `HistGradientBoostingClassifier` with `class_weight='balanced'`.

The model's predictive power is concentrated in driver-assignment signals (`Avg VTAT`, `vtat_missing`). Features known only after ride completion (booking value, ride distance, ratings, payment method) were explicitly excluded to prevent data leakage.

### 1. Clone the repository

```bash
git clone <your-repo-url>
```

### 2. Create and activate a virtual environment


## Running the Flask API

The model must be trained first (run the notebook through Section 10).

```bash
python src/app.py
```

The API runs at `http://localhost:5000`.

### Endpoints

**GET `/health`**
```bash
curl http://localhost:5000/health
# {"model": "HistGradientBoosting", "status": "ok"}
```

**POST `/predict`**
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "hour": 22,
    "month": 8,
    "is_weekend": 1,
    "vtat_missing": 0,
    "Avg VTAT": 12.5,
    "hour_bucket": "evening",
    "day_of_week": "Friday",
    "Vehicle Type": "Go Sedan",
    "pickup_grouped": "Other",
    "drop_grouped": "Other"
  }'
# {"predictions": [{"cancelled": 1, "cancellation_probability": 0.712}]}
```

### Input fields

| Field | Type | Description |
|---|---|---|
| `hour` | int | Hour of booking (0–23) |
| `month` | int | Month of booking (1–12) |
| `is_weekend` | int | 1 if Saturday or Sunday, else 0 |
| `vtat_missing` | int | 1 if no driver was found (Avg VTAT is null) |
| `Avg VTAT` | float | Vendor time to assign in minutes (null if no driver found) |
| `hour_bucket` | str | `night`, `morning`, `afternoon`, or `evening` |
| `day_of_week` | str | Full day name, e.g. `Monday` |
| `Vehicle Type` | str | e.g. `Auto`, `Go Sedan`, `eBike` |
| `pickup_grouped` | str | Pickup location name or `Other` |
| `drop_grouped` | str | Drop location name or `Other` |

## Ethical Considerations

- **Bias:** The model uses pickup and drop location as features. If certain areas correlate with demographic groups, the model may produce systematically different cancellation predictions by neighbourhood. This should be audited before deployment.
- **Privacy:** The dataset contains Customer IDs. These are anonymised in the training data but any production deployment must ensure personally identifiable information is not logged or exposed through the API.
- **Misuse:** Cancellation probabilities could be used to deprioritise certain bookings or locations, potentially reducing service availability for already underserved areas.
- **Transparency:** Drivers and customers affected by platform interventions triggered by this model should have access to an explanation of why they were flagged.
